\subsubsection{1 часть - ранжирование}
\begin{itemize}
\item Определена область исследования: направленные ацикличные графы зависимости задач (DAG). В каждом графе присутствует финальный узел "finish", означающий завершение процесса выполнения задач.
\item Получены 3 алгоритма ранжирования узлов графов зависимостей задач по влиянию на остальные задачи:
\begin{enumerate}
\item simpleRank - алгоритм начинается с последней вершины (всегда с весом 1) и для каждого узла i рассчитывает вес как сумму весов всех исходящих узлов + 1
\item successorsRank - алгоритм рассчитывает вес пропорционально количеству всех зависимых задач
\item pageRank - алгоритм основан на поиске собственных векторов P-матрицы для собственного значения 1. Для невырожденности матрицы граф зацикливается -  финальный узел соединяется с начальными. Эта ситуация соответствует последовательному выполнению подобных процессов.
\end{enumerate}
\item Все алгоритмы, предположительно, дают топологическую сортировку - TODO исследовать
\item Алгоритмы дают разную сортировку
\item Вставить данные стравнения алгоритмов на графах 10, 100, 1000 узлов TODO
\end{itemize}


\subsubsection{2 часть - определение оптимальной стратегии исполнения}
\begin{itemize}
\item Реализована симуляция выполнения процесса по графу зависимостей в однопоточной среде со следующими параметрами:
\begin{itemize}
\item Время выполнения задачи
\item Вероятность возникновения ошибки
\item Время исправления ошибки (опционально, считается >> времени выполнения процесса)
\end{itemize}
\item  Введено понятие покрытия процесса: среднее отношение количества выполненных задач к количеству задач в процессе в результате симуляции
\item При условии время исправления ошибки >> времени выполнения процесса среднее покрытие не зависит от выбора топологической сортировки и может быть рассчитано аналитически через вероятности возникновения ошибок.
\end{itemize}

\subsubsection{Дополнительно}
\begin{itemize}
\item Добавление дополнительных факторов - вероятность падения ошибки, разные типы зависимостей (условная зависимость, "Finish to start", "Start to start", etc)
\item Обобщение задачи на многопоточную систему - минимизирование времени простоя (без перераспределения задач между потоками)
\item Обобщение задачи на многопоточную систему - алгоритмы перераспределения задач при возникновении ошибки
\end{itemize}
